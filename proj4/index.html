Part 1

In this section, we implemented the algorithm to build the grid of masses of springs. We created the evenly distributed grid of masses by simple arithmetic manipulation of small offsets and remembered to include the z random offset.

Next, we created the springs to apply the three constraints. In particular, we iterated through all the points and checked for edge cases, and applied the constraints through index manipulations according to what the spec describes.

Part 2

Implementation

For part 2, we first summed up all the forces acting on the masses and applied the force to the masses using Newton’s law. After this, I applied the spring’s forces using the given equation.

Next, we followed the equation for Verlet Integration to derive the position of each mass using the previous position, velocity, acceleration, as well as the damping term. Finally, we made sure that the springs do not go beyond 10% greater than its rest_length by applying a correction term once they do exceed 10%. 

Experimenting With Parameters:

Higher ks makes the cloth more rigid because the springs are harder to be stretched. Therefore, the wrinkles on the cloth are sparse and larger. Smaller ks, however, makes the wrinkles smaller, since the cloth begins to bend.

Part 3

Implementation:

Given a point, we handled plane-sphere collision by first testing if the point is inside the sphere. If that is the case, then we calculate where the point should have intersected the sphere, compute the correction vector and then update the point's position adjusted by the correction vector and scaled by friction. Similarly, for plane-plane collision, we test if the point and the plane's normal is in the same direction. If not, then the point is under the plane and we need to compute a correction vector. We calculated this vector using the ray-plane intersection formula given in the lecture and then applied an additional offset. Finally, we update the point's position adjusted by the correction vector and scaled by friction.

Difference in different spring constants (ks):

A larger spring constant makes the cloth more rigid; the cloth doesn't fully drape over the sphere because the "internal springs" of the cloth are able to better overcome gravity. In comparison, a smaller spring constant makes the cloth softer, allowing the cloth to conform to the sphere's shape and fully drape over it. Since the force between the "internal springs" of the cloth are small, the cloth becomes more wrinkled compared to a large spring constant.

Part 4

Implementation:

We used a hash function similar to Java's hashCode() function to compute a unique float given a point's coordinates.

To construct the spatial map, we calculate the hash of the input point. If the hash doesn't exist in the map, we initialize the hash to map to a vector of points. Otherwise, we append the point to the hash's corresponding bucket.

Given a point, we implemented self-collision by first calculating its hash value. Then we initialize the total correction vector to be zero value. For each point in the hash bucket, we test if the point and the given point are less than 2 * thickness apart. If they are too close, we calculate the correction vector needed such that the points are 2 * thickness apart, and add this new correction vector to the total correction vector. When all points in the bucket have been computed, we average the total correction vector and scale down by the simulation steps.

Part 5

Shader programs are specialized programs that run in parallel on the GPU. They take inputs such as textures, color, camera position, vertex position, etc. and output a four-dimensional vector. Vertex shaders apply transforms to vertices and can modify properties such as their position and normals. Fragment shaders take in fragments from the vertex shader and compute a pixel color. The output of the vertex shader goes into the fragment shader.

The Blinn-Phong model is composed of three parts: ambient, diffuse and specular. In our implementation, the ambient component is simply a constant. The diffuse component and specular components are computed using the formulas provided in the spec. The diffuse component gives the material a matte appearance, while the specular component gives the material smooth highlights. Finally, we can adjust the material's brightness using the ambient component.

Implementation:

We implemented diffuse and phong shaders as described above. For the texture shader, we simply sampled the input texture at the given coordinates. For the bump and shader, we calculate the displaced world space normal and summed up all other shading effects to return. In terms of the displacement shader, in addition to the code we implemented in .frag for bump effects, we also displaced the vertex positions in the direction of the model normal in .vert file. For the mirror shader, we found the out-going camera ray w_o, mirrored it across the surface normal to get w_i and sampled the environment map using w_i.

The two approaches are the same except displacement simulation also offsets the vertices’ positions. The two simulations here have the same parameters, same shapes, and same textures, and thus the visual effects look roughly the same. However, by paying attention to the peripherals of the ball, we can notice that the latter displaced ball has a rough and uneven peripheral which aligns with the visual bumps of the texture.

Coarseness 16,  displacement

Coarseness 16,  bump

Coarseness 128,  bump

Coarseness 128,  displacement

The two approaches react differently to the mesh’s coarseness. Bump simulations do not react much to the differences in the coarseness, so no matter what the coarseness is, the visual effects are pretty similar. This might be due to the fact that bump simulation does not reflect the underlying supporting structures of the sphere as much as displacement simulation does. However, for displacement simulations, the higher the coarseness, the more refined the evenness looks, and the more real it is. This is possibly because the high-quality spheres allow for more refined and dense calculations of the vectors in response to the offsets (displacements).
